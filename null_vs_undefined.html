Null Vs Undefined

basically both mean that there is no value. Infact equating both using loose equality sign == will return true. However, using strict equality === will return false.

So what really are their differences?

Null

When a variable is null it means that the variable has no value and it was explicitly set to have null value by the programmer. This means a variable will never be null unles set to null in the code.

For example, in a find function that queries a database for an entry. If no entry exists it makes the most sense to return null since you are stating that there is no value found.

Undefined

?Undefined means there is no value because no value has been set. If you create a variable and do not assign it a value, then it will be null.

What now? When do I use these?

Essentially you could set a variable to undefined when trying to convey a message that the variable no longer contains any useful information while when the value is null you are specifically saying the result of some action has no value
